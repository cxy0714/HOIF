# HOIF Package File Structure

## File Organization

```
HOIF/
├── R/
│   ├── hoif.R           # Main analysis functions + ustat
├── tests/
├── man/                 # Auto-generated by roxygen2
├── DESCRIPTION
├── NAMESPACE           # Auto-generated by roxygen2
└── README.md
```

## File Contents

### R/hoif.R
Contains all the main statistical functions:
- `transform_covariates()`
- `compute_residuals()`
- `compute_gram_inverse()`
- `compute_basis_matrix()`
- `ustat()` - the Python interface function
- `expr_list_to_einstein()` - helper for ustat
- `compute_hoif_estimators()`
- `hoif_ate()` - main function
- `print.hoif_ate()`
- `plot.hoif_ate()`

### R/setup.R
Contains environment setup functions:
- `check_python_env()` - internal checker
- `setup_hoif()` - user-facing setup
- `check_hoif_setup()` - user-facing status check

### R/package.R
Contains package-level documentation:
- Package environment definition
- `.onLoad()` function
- Package documentation (`?HOIF`)

## Why This Structure?

1. **hoif.R** = "What users want to do" (analyze data)
2. **setup.R** = "How to prepare the environment" (setup Python)
3. **package.R** = "Package metadata and startup" (documentation)

## No zzz.R Needed

The `.onLoad()` function goes in `package.R` instead. The traditional `zzz.R` 
was just a convention for startup code, but it's cleaner to put it in 
`package.R` with the package documentation.
